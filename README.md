# Network Topology Viewer (NTV) 🌐

Привет! Это интерактивный дашборд для мониторинга топологии сети и состояния сетевых устройств в режиме реального времени. Проект создан для визуализации работы сетевой инфраструктуры (NOC — Network Operations Center). Он показывает, как связаны устройства, какие из них перегружены, и выводит уведомления об инцидентах.

---

## 🏗️ Как устроен проект (Архитектура)

Проект разделен на две основные части:

1. **Frontend (`/frontend`)** — визуальная панель управления:
   - **React + TypeScript + Vite** — быстрый интерфейс.
   - **@xyflow/react (React Flow)** — отображает интерактивную карту сети (устройства, линки, их загруженность).
   - **Reaviz** — рисует графики изменения метрик (CPU, RAM, температура) в реальном времени.
   - **Ant Design** — используется для компонентов интерфейса (вкладки, кнопки, модалки).
   - **Sass (SCSS) Modules** — изоляция стилей для каждого компонента.
   - **Redux Toolkit Query** — управление состоянием и работа с кэшем данных.

2. **Backend (`/backend`)** — сервер симуляции:
   - **Node.js + Express + TypeScript** — веб-сервер.
   - **WebSocket (ws)** — отправляет ежесекундные обновления метрик сети и мгновенно доставляет уведомления об авариях на фронтенд.
   - **Симулятор телеметрии** — раз в секунду обновляет метрики устройств (случайный дрейф) и раз в 12 секунд генерирует случайные сетевые проблемы для демонстрации работы алармов.

## 🚀 Быстрый старт

### 1. Подготовка
Убедитесь, что у вас установлен **Node.js** (версии 20 или выше).

### 2. Установка всех зависимостей
В корне проекта есть удобный скрипт, который установит всё одной командой для корня, бэкенда и фронтенда:
```bash
npm run install:all
```

### 3. Запуск в режиме разработки
Запустить одновременно бэкенд и фронтенд локально можно одной командой из корня:
```bash
npm run dev
```
После этого:
* Интерфейс откроется по адресу: **`http://localhost:3000`**
* WebSocket/API-сервер запустится на порту: **`3001`**

## 🛠️ Полезные команды

Если вам нужно управлять частями проекта по отдельности, используйте следующие команды из корня:

* **Запуск бэкенда отдельно:**
  ```bash
  npm run start:backend
  ```
* **Запуск фронтенда отдельно:**
  ```bash
  npm run start:frontend
  ```
* **Проверка кода линтером (Biome):**
  ```bash
  npm run lint
  ```
* **Автоматическое исправление ошибок форматирования:**
  ```bash
  npm run lint:fix
  ```

## 🤖 CI/CD (Автоматизация)

В проекте настроен пайплайн **GitLab CI** (`.gitlab-ci.yml`), который при каждом коммите автоматически:
1. Скачивает и кэширует зависимости (`node_modules`).
2. Проверяет качество кода линтером (`Biome`).
3. Собирает production-версии фронтенда и бэкенда для тестирования стабильности.
