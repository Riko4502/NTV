# Network Topology Viewer (NTV)

Привет! Это интерактивный дашборд для мониторинга топологии сети и состояния сетевых устройств в реальном времени. Проект создан для визуализации работы сетевой инфраструктуры (NOC — Network Operations Center). Он показывает, как связаны устройства, какие из них перегружены, и выводит уведомления об инцидентах.

---

## Как устроен проект (Архитектура)

Проект разделён на две основные части:

1. **Frontend (`/frontend`)** — визуальная панель управления:
   - **React + TypeScript + Vite** — быстрый интерфейс.
   - **@xyflow/react (React Flow)** — отображает интерактивную карту сети (устройства, соединения, их загруженность).
   - **Reaviz** — строит графики изменения метрик (CPU, RAM, температура) в реальном времени.
   - **Ant Design** — набор компонентов интерфейса (вкладки, кнопки, модальные окна).
   - **Sass (SCSS) Modules** — изоляция стилей для каждого компонента.
   - **Redux Toolkit Query** — управление состоянием и кэширование данных.

2. **Backend (`/backend`)** — сервер симуляции:
   - **Node.js + Express + TypeScript** — веб‑сервер.
   - **WebSocket (ws)** — отправляет каждую секунду обновления метрик и мгновенно доставляет уведомления об авариях на фронтенд.
   - **Симулятор телеметрии** — раз в секунду обновляет метрики устройств (случайный дрейф) и раз в 12 секунд генерирует случайные сетевые проблемы для демонстрации работы тревог.

## Быстрый старт

### 1. Подготовка
Убедитесь, что установлен **Node.js** версии 20 или выше.

### 2. Установка зависимостей
В корне проекта есть скрипт, который установит все зависимости для корня, бэкенда и фронтенда одной командой:
```bash
npm run install:all
```

### 3. Запуск в режиме разработки
Запустить бэкенд и фронтенд одновременно можно одной командой из корня:
```bash
npm run dev
```
После запуска:
* Интерфейс будет доступен по адресу `http://localhost:3000`.
* API и WebSocket‑сервер запустятся на порту `3001`.

## Полезные команды

Для работы с отдельными частями проекта используйте следующие команды из корня:

- **Запуск бэкенда отдельно:**
  ```bash
  npm run start:backend
  ```
- **Запуск фронтенда отдельно:**
  ```bash
  npm run start:frontend
  ```
- **Проверка кода линтером (Biome):**
  ```bash
  npm run lint
  ```
- **Автоматическое исправление форматирования:**
  ```bash
  npm run lint:fix
  ```

## CI/CD (Автоматизация)

В проекте настроен пайплайн **GitLab CI** (`.gitlab-ci.yml`). При каждом коммите он автоматически:
1. Скачивает и кэшироует зависимости (`node_modules`).
2. Проверяет качество кода с помощью линтера (`Biome`).
3. Собирает production‑версии фронтенда и бэкенда для проверки стабильности.
